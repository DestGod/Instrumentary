		СПРАВОЧНИК
$ ls 
посмотреть список директорий и файлов в текущей папке

$ cd 'путь'
переход в папку по пути (можно локальный и абсолютный)

$ git config 
конфигурационный файл

$ git init
инициализация гита в текущей директории

$ git remote add 'псевдоним' 'url'
добавление удаленного подключения

$ git remote -v
посмотреть удаленные подключения

$ git add 'название файла или .'
добавления файла в индекс для последующего коммита

$ git status
посмотреть статус файлов в индексе

$ git commit -m 'сообщение коммита'
собственно сам коммит с коменатрием

$ git push 'псведоним' 'ветка'
пуш коммита на удаленный репозиторий 



		ПЕРВЫЙ ЗАПУСК
	1. в кавычках вводим свое имя или ник
$ git config --global user.name "John Doe"
	2. почта
$ git config --global user.email johndoe@example.com

		СОЗДАНИЕ СВОЕГО ПРОЕКТА
	1. заходим на гитхаб в аккаунт, нажимаем на аватарку и переходим на вкладку 'Your repositories'
	2. жмем на зеленую кнопку 'New', заполняем поле 'Repository name', можем поменять доступ - Public/Private, жмем на 'Create repository'
	3. создаем папку со своим проектом (проект в этом случае - общее слово, здесь может быть проект в виде веб-сайта или проект
	на каком-нибудь C++), если это юнити-проект, то делаем эту папку с помощью нее
	4. создаем файлик ".gitignore" и вписываем туда папки/файлы, которые нам точно не надо будет пушить на удаленный репозиторий,
	в случае с юнькой это -
# Unity
/[Ll]ibrary/
/[Tt]emp/
/[Oo]bj/
/[Bb]uild/
/[Bb]uilds/
/Assets/AssetStoreTools*
sysinfo.txt
*.pidb.meta
 
# VS/Rider/MD/Consulo
ExportedObj/
.consulo/
.idea/
*.csproj
*.unityproj
*.sln
*.suo
*.tmp
*.user
*.userprefs
*.pidb
*.booproj
*.svd
 
# Builds
*.apk
*.unitypackage
*.app
*.exe
	5. запускаем Git Bash:
		а. пкм в папке с проектом->'Git Bash here'
		б. просто запускам .exeшник и прописываем абсолютный путь до папки с проектом с помощью команды 'cd', например
$ cd D:/'Новая папка'/Project
	6. после этого надо убедиться, что вы находитесь в папке проекта. над курсором будет строка по типу, в конце должен быть верный путь
	Username@DESKTOP-SV32LS MINGW64 /d/Новая папка/Project
	$
	7. вводим команду для иницилазицаии гита в этой папке
$ git init
	8. после этого в конце строки над курсором должна появиться надпись (master) и в папке с проектом должна появиться скрытая папка .git
	9. создаем дистанционное (удаленное) подключение к нашему созданному репозиторию на гитхабе командой ('sampleproject' меняем на 
	любое слово на латинице которое вам удобно, это что-то вроде сокращенного псведонима подключения, ссылку также меняем на 
	свой репозиторий)
$ git remote add sampleproject https://github.com/username/project
	10. проверяем подключение командой (ответ должен быть в виде двух строчек, в которых написан псевдоним подключения, ссылка и в конце
	еще fetch и push)
$ git remote -v 
	11. пишем команду, чтобы добавить '.gitignore' в локальный коммит и ждем, пока все сделается
$ git add .gitignore
	12. проверяем файлик командой (должен быть список файлов с надписью перед ними 'new file:')
$ git status
	13. делаем коммит с комментарием в кавычках (в нашем случае 'adding files to repository')
$ git commit -m 'added gitignore'
	14. пушим наш коммит на удаленный репозиторий (вписываем вместо 'sampleproject' тот псевдоним, который указали в 9 пункте)
$ git push sampleproject master
	15. создаем файлики в проекте (если сделали его через юньку, то переходим к некст пункту)
	16. пишем команду, чтобы добавить все файлы в локальный коммит и ждем, пока все сделается
$ git add .
	17. переходим на пункт 12
